## Context
需要重新設計 UI 佈局，從單欄垂直堆疊改為專業的左右分欄佈局。左側是影片預覽和播放控制，右側是片段管理和輸出設定。這種佈局更符合專業影片編輯工具的使用習慣，能更有效地利用螢幕空間。

## Goals / Non-Goals
- Goals:
  - 實作左右分欄佈局（左側 65-70%，右側 30-35%）
  - 左側：影片預覽視窗（最大化高度）、播放控制、進度條
  - 右側：片段列表（表格形式）、新增片段按鈕、輸出設定、批次輸出按鈕
  - 頂部：標題列（Logo/標題、載入影片、設定、幫助）
  - 底部：狀態列（顯示當前狀態）
  - 片段列表改為表格形式，顯示編號、名稱、開始時間、結束時間、操作

- Non-Goals:
  - 不實作時間軸視覺化（留待後續提案）
  - 不實作鍵盤快捷鍵（留待後續擴充）
  - 不實作多視窗模式（單一視窗設計）
  - 不實作主題切換（留待後續擴充）

## Decisions
- Decision: 使用 CSS Grid 或 Flexbox 實現分欄佈局
  - Alternatives considered: 絕對定位、固定寬度
  - Rationale: Grid/Flexbox 提供更好的響應式控制和靈活性

- Decision: 左側預覽視窗佔據 65-70% 寬度
  - Alternatives considered: 50/50、70/30
  - Rationale: 預覽視窗是主要工作區域，需要更多空間

- Decision: 片段列表使用表格形式
  - Alternatives considered: 卡片式、列表式
  - Rationale: 表格形式更緊湊，能顯示更多資訊，符合專業工具習慣


## Risks / Trade-offs
- Risk: 響應式設計在小螢幕上可能顯示不佳 → Mitigation: 設定最小寬度，考慮後續加入響應式斷點
- Risk: 表格列表在片段很多時可能過長 → Mitigation: 後續可考慮虛擬化列表或分頁
- Risk: 佈局改變可能影響現有功能 → Mitigation: 逐步遷移，確保每個功能都正常運作

## Migration Plan
1. 先建立新的佈局結構（Header、StatusBar）
2. 將現有元件遷移到新佈局
3. 優化各元件的樣式和尺寸
4. 測試所有功能在新佈局下是否正常
