## Context
目前的播放器在播放結束時會自動停止。使用者需要重複播放時，必須手動重新點擊播放按鈕。循環播放功能可以自動重複播放，提升使用體驗。

## Decisions
- **狀態管理**：在 `videoStore` 中新增 `loop` boolean 狀態，使用 Zustand 管理，符合專案現有模式。
- **按鈕位置**：放置在播放控制區，時間軸下方，與播放/暫停按鈕同一行，方便操作。
- **循環邏輯**：
  - 監聽 video 元素的 `ended` 事件
  - 當 `loop` 為 `true` 且播放結束時：
    - 一般模式：`video.currentTime = 0`，然後 `video.play()`
    - 預覽模式：`video.currentTime = currentPreviewSegment.startTime`，然後 `video.play()`
- **與預覽模式的協同**：
  - 預覽模式下，循環應該在片段範圍內循環
  - 需要確保循環邏輯在預覽模式的播放範圍限制之後執行，或者整合在一起

## Risks / Trade-offs
- **效能**：循環播放會持續消耗資源，但這是預期行為。
- **與預覽模式的衝突**：預覽模式有播放範圍限制（到達結束時停止），需要與循環邏輯協調。解決方案：在循環啟用時，不執行自動停止，而是執行跳轉。
